# UX 文案框架

## 5 个原则

### 1. 清晰（Clear）

用户不应该需要猜这段文字是什么意思。避免行话、缩写、双关语。

### 2. 简洁（Concise）

每个词都有存在的理由。能用 5 个字说清楚的，不要用 10 个字。

### 3. 一致（Consistent）

同一个操作在不同地方用同一个词。"删除"就是"删除"，不要有时叫"移除"有时叫"清除"。

### 4. 有用（Useful）

文案要帮用户做决定或完成任务，不只是填充空间。

### 5. 有人味（Human）

像和朋友说话一样，不像机器人或法律文书。

## 文案模式

### 按钮 / CTA

**原则：**

- 动词开头，描述点击后会发生什么
- 具体比模糊好
- 和上下文匹配

| 场景     | 差       | 好           |
| -------- | -------- | ------------ |
| 提交表单 | 确定     | 提交订单     |
| 注册     | 提交     | 免费注册     |
| 删除     | 确认     | 删除这条记录 |
| 升级     | 了解更多 | 升级到专业版 |

### 错误提示

**公式：** 发生了什么 + 为什么 + 怎么解决

| 差        | 好                                       |
| --------- | ---------------------------------------- |
| 错误！    | 手机号格式不正确，请输入 11 位手机号     |
| 请求失败  | 网络连接中断，请检查网络后重试           |
| 无效输入  | 密码至少需要 8 个字符，包含数字和字母    |
| Error 500 | 服务暂时不可用，我们正在修复，请稍后再试 |

**注意：**

- 不要责怪用户（"你输入错误"→"格式不正确"）
- 不要用技术术语（Error 500、Invalid Token）
- 具体说明怎么修复

### 空状态

**公式：** 这是什么 + 为什么是空的 + 怎么开始

| 场景       | 差         | 好                                   |
| ---------- | ---------- | ------------------------------------ |
| 没有消息   | 暂无数据   | 还没有消息，新消息会出现在这里       |
| 搜索无结果 | 未找到结果 | 没有找到"XX"的结果，试试换个关键词？ |
| 空购物车   | 购物车为空 | 购物车还是空的，去逛逛热门商品？     |

### 确认对话框

**原则：**

- 标题说清楚要确认什么
- 说明后果（尤其是不可逆操作）
- 按钮用具体动词，不要"确定/取消"

**模板：**

```
标题：删除这个项目？
说明：删除后无法恢复，项目中的所有文件也会被删除。
[取消]  [删除项目]  ← 危险操作用红色按钮
```

| 差                          | 好                                            |
| --------------------------- | --------------------------------------------- |
| 确定要操作吗？[确定] [取消] | 删除"我的项目"？此操作无法撤销。[取消] [删除] |

### 加载状态

| 场景               | 文案                         | 补充                 |
| ------------------ | ---------------------------- | -------------------- |
| 快速加载（<2秒）   | 不需要文案，用动画即可       | 骨架屏或微动画       |
| 中等加载（2-5秒）  | 加载中...                    | 可加进度指示         |
| 长时间加载（>5秒） | 正在生成报告，大约需要 30 秒 | 给出预估时间         |
| 后台处理           | 文件正在处理，完成后会通知你 | 允许用户继续做其他事 |

### 工具提示（Tooltip）

- 不超过一行（约 20 个字）
- 解释"是什么"或"为什么"，不重复按钮上已有的文字
- 不要把关键信息藏在 tooltip 里

### 引导/Onboarding 文案

- 聚焦价值，不是功能："管理你的日程"而非"日历功能"
- 每步只说一件事
- 允许跳过
- 用"你"不用"用户"

## 语气适配

| 场景   | 语气         | 例子                             |
| ------ | ------------ | -------------------------------- |
| 成功   | 积极、简短   | 保存成功！                       |
| 错误   | 冷静、有帮助 | 上传失败，文件大小不能超过 10MB  |
| 警告   | 认真、不吓人 | 这个操作会影响所有团队成员       |
| 等待   | 耐心、有进度 | 正在导出，大约还需要 10 秒       |
| 空状态 | 友好、引导   | 还没有收藏，点击心形图标开始收藏 |

## 中文 UX 文案注意事项

- 避免直译英文（"Got it"不要翻译成"得到了"，用"知道了"或"好的"）
- 中文不需要首字母大写的概念，但要注意全角/半角标点一致
- 数字用阿拉伯数字（"3 个文件"而非"三个文件"）
- 保持主语一致（要么全用"你"，要么全省略主语）
