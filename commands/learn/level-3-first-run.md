# 第3关：实战写作

## 目标

让用户亲手运行一个 Skill，理解完整的执行流程。

## 教学流程

### 1. 选择主题

使用 AskUserQuestion 提供 4 个可选主题：

```
你想聊什么话题？选一个最感兴趣的：

1. 转行/跨界做新领域的心路历程
2. 第一次用 AI 工具完成工作任务
3. 一次跨部门协作中的意外收获
4. AI 正在改变我的工作方式
```

记录用户选择的主题。

### 2. 工作空间说明

> 接下来我会直接启动访谈写作流程，先告诉你文件会保存在哪：
>
> ```
> 📂 文件位置：
>   访谈记录 → writing/drafts/interview-transcript.md
>   最终文章 → writing/output/日期-标题.md
> ```
>
> 你不需要手动操作这些文件，Skill 会自动处理。
>
> 这是教学体验版，访谈会比正式版精简一些，让你快速走完全流程。

### 3. 记录执行前状态

先记录当前 `./writing/output/` 下的文件数量：

```bash
ls ./writing/output/ | wc -l
```

保存这个数字，后面验证用。

### 4. 直接启动访谈写作流程

**不要让用户退出再手动输入命令。** 选完主题后，直接在当前会话中执行访谈和写作。

1. 读取 `commands/interview-write/style-guide.md` 获取写作规范
2. 按以下流程执行（**不需要读取 interview-write 的 SKILL.md**，用下方的内联规则即可）：
   - 访谈：按下方的教学模式访谈规则执行，实时保存到 `drafts/interview-transcript.md`
   - 写作：读取 `./writing/examples/` 学习风格 → 写作 → 自动优化 → 保存到 `writing/output/` 并展示

整个访谈和写作在当前会话内完成，用户无需退出。

**教学模式访谈控制（轮次少，质量高）：**

访谈控制在 **5-8 轮**，但每一问必须精准、有穿透力。轮次少不等于浅，而是每一轮都要挖到真东西。

**素材目标清单——访谈结束前必须收集齐 4 样：**

1. **一个具体场景**（时间、地点、当时在做什么——让读者能"看见"）
2. **一个决策/转折点**（在那个场景下，你做了什么选择？为什么？）
3. **一个情感冲突**（内心纠结、矛盾、意外、不甘——写作的燃料）
4. **一句反思/感悟**（回头看，你怎么理解这件事——文章的升华点）

**随意回答应对：**

- 如果用户回答 < 20 字且缺乏具体场景/细节，撬动一次：「能再具体一点吗？比如当时是什么情况？」
- 撬动后用户仍然简短，接受并继续下一问，不要反复追问
- 每次访谈最多撬动 2 次

**素材检查点（第 4 轮后触发）：**

- 内部盘点 4 样素材的收集进度
- 缺失的项在后续提问中针对性补齐
- 如果到第 6 轮仍只收集到 2/4，提议结束访谈并告知用户"素材有点少，写作时我会适当补充一些细节"

**素材兜底（写作前）：**

- 如果访谈结束时素材清单只集齐 2/4，AI 在写作阶段基于已有素材合理补充细节，确保文章有完整的故事弧线
- 不需要标注哪些是补充的，保持文章自然流畅

**高密度提问技巧：**

- **复合追问**：不要把事实和情感拆成两轮问。一轮就要打穿：「你刚说 XX，当时心里什么感觉？犹豫过吗？」
- **锐利切口**：不问泛泛的"为什么"，直接切最有张力的点：「你说得很平静，但放弃之前积累的东西，真的没有不甘心吗？」
- **抓住金句信号**：用户随口说出的比喻、反常表达、自嘲，往往就是金句原料。立刻追问放大：「你刚才说'像是重新活了一次'——这个感觉能再展开说说吗？」
- **用对比制造深度**：「做这个决定之前和之后，你觉得自己最大的变化是什么？」
- **意外感追问**：找到故事里"不合常理"的部分追问，这些往往是最好的写作素材

**节奏参考：**

- 第 1-2 轮：锁定具体场景 + 关键决策（事实层，但追问时顺带触碰情感）
- 第 3-4 轮：深挖情感冲突（这是核心，必须问出"从没说过的话"）
- 第 5 轮：收一个有力的反思/感悟，如果前面已经自然涌现就直接收尾

**收尾判断：** 当素材清单 4 样都集齐，且至少有 1 个强情感点（用户回答时出现停顿、自嘲、感叹等信号），就可以结束访谈

> **提示：** 访谈记录会自动保存到 drafts/interview-transcript.md，第4关会复用这份素材，不用重新聊一遍。

### 5. 复盘 — 终端展示终稿

写作完成后，**读取文件内容并在终端直接展示全文**：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📄 你的第一篇 Skill 产出
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

{读取并展示文章全文}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📁 已保存到：writing/output/{文件名}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

简要点评：

- 指出文章中的金句（加粗部分）
- 指出风格参考的痕迹
- 说明这篇文章是怎么从访谈内容转化来的

## 验证

1. `./writing/output/` 下有新的 .md 文件（文件数比执行前多）
2. 新文件字数 > 200 字符

通过后显示：

```
✅ 第3关通过！你已经成功运行了第一个 Skill，并产出了一篇文章。
```

然后执行通关总结（见 summaries.md）。

通关总结之后，追加进阶提示：

> 💡 **进阶提示：**
> `writing/examples/` 里的文章决定了 AI 的写作风格。
> 替换成你自己喜欢的文章（3-5 篇 .md），下次运行就会用新风格。
> AI 学的是风格和节奏，不是字数——参考文章可以比输出上限长。

## 未通过处理

如果写作流程中断或产出不完整，按以下优先级处理：

1. **有访谈记录但文章未生成**（`drafts/interview-transcript.md` 存在且 > 200 字符）：

   > 访谈素材已经够了，我直接从写作阶段开始。
   > 从 Phase 4（学习风格）继续执行，跳过重新访谈。

2. **访谈记录也没有或不完整**：
   使用 AskUserQuestion 提问：
   - "重新开始访谈"（再来一次，但会更精简）
   - "用示例素材快速体验"（跳过访谈，用预设内容生成一篇演示文章，让用户先看到完整流程）

   如果选"用示例素材"：用主题的通用素材（基于用户选的话题方向）模拟一份简短访谈记录，然后从 Phase 4 继续。
