# 第2关：看懂设计系统

## 目标

理解设计系统输出的每个部分，能判断"哪里可能不适合我的项目"。这是后续关卡（选择、评审）的基础。

## 教学流程

### Step 1 — 打开第 1 关的设计系统输出

读取 `./design/output/learn-progress.md` 获取案例项目信息，然后重新生成（或从缓存读取）第 1 关的精准版设计系统输出。

### Step 2 — 逐块讲解

将设计系统拆成 7 个模块，每个用 2-3 行讲清楚。不要一次性全展示，每讲完 2 个模块用 AskUserQuestion 过渡。

| 模块             | 含义                                              | 影响什么               |
| ---------------- | ------------------------------------------------- | ---------------------- |
| **PATTERN**      | 页面模式（如 Hero + Features + CTA）              | 页面整体结构和浏览路径 |
| **STYLE**        | 视觉风格（如 Glassmorphism、Minimalism）          | 整体视觉感受           |
| **COLORS**       | 配色方案（Primary/Secondary/CTA/Background/Text） | 品牌调性和情感传达     |
| **TYPOGRAPHY**   | 字体搭配（Heading + Body）                        | 阅读体验和品牌个性     |
| **KEY EFFECTS**  | 动效建议（如 blur、transition timing）            | 交互感受               |
| **AVOID**        | 反模式（本项目不应该做的事）                      | 避免常见错误           |
| **PRE-DELIVERY** | 交付前检查清单                                    | 上线前的专业度保障     |

### Step 3 — 8 个优先级规则速览

简要介绍设计规则的优先级体系（不超过 8 行）：

```
CRITICAL → 无障碍（对比度、alt text）+ 触摸交互（44×44px）
HIGH     → 性能（图片优化）+ 响应式（viewport）
MEDIUM   → 字体/配色/动效/风格选择
LOW      → 图表和数据可视化
```

告诉用户：不需要记住所有规则，但要知道**优先级顺序**——CRITICAL 的问题必须修，LOW 的可以后面再优化。

### Step 4 — 实战练习：找问题

这是本关的核心互动。AI 从第 1 关的设计系统输出中，预设 3 个"可能不适合"的点，用 AskUserQuestion（multiSelect: true）让用户判断：

**问题设计原则**：

- 根据用户选的案例项目，找出设计系统中**合理但不完美**的推荐
- 不是"明显错误"，而是"需要人判断"的地方

**示例（咖啡品牌案例）**：

> 看看第 1 关生成的设计系统，你觉得哪些地方可能不太适合你的咖啡品牌？

选项（multiSelect）：

- 配色方案偏冷色调，和"温暖手工"的品牌感不一致
- 推荐的字体太正式/太技术感，缺少手工/文艺气质
- 反模式列表缺少"不要用库存照片"这类品牌相关的避讳
- 都挺好的，我觉得没问题

**无论用户怎么选，都给正反馈**：

- 选了问题 → "好眼光！你发现了 AI 推荐的局限——它按产品类型匹配，但不了解你的品牌个性。"
- 觉得没问题 → "AI 的推荐确实不错，但第 3 关你会有机会亲自选择更贴合品牌的方案。"

### Step 5 — 通关总结

**核心教学点**：看懂输出 → 才能判断哪里需要改 → 这就是设计师的价值。

## 验证条件

- 用户完成了 Step 4 的"找问题"练习（AskUserQuestion 已回答）
