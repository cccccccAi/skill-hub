# 竞品分析框架

## 竞品识别

### 4 类竞品

| 类型         | 定义                 | 例子（以在线教育为例） |
| ------------ | -------------------- | ---------------------- |
| **直接竞品** | 同品类、同用户群     | 猿辅导 vs 作业帮       |
| **间接竞品** | 不同方式解决同一问题 | 在线课 vs 线下补习班   |
| **相邻竞品** | 可能进入你市场的玩家 | 抖音做知识付费         |
| **替代品**   | 用户不解决问题的选项 | 自学、问同学           |

**技巧：** 不要只盯着直接竞品。相邻竞品和替代品往往是真正的威胁。

## 功能对比矩阵

```
| 功能 | 我们 | 竞品 A | 竞品 B | 竞品 C |
|------|------|--------|--------|--------|
| [功能 1] | ✅ 完善 | ✅ 基础 | ❌ 无 | ✅ 完善 |
| [功能 2] | ⚠️ 开发中 | ✅ 完善 | ✅ 完善 | ❌ 无 |
| [功能 3] | ❌ 无 | ❌ 无 | ✅ 独有 | ❌ 无 |
```

**评分标准：**

- ✅ 完善：功能成熟，用户体验好
- ✅ 基础：有此功能但体验一般
- ⚠️ 开发中：已规划/开发中
- ❌ 无：没有此功能

**注意：** 不要只比功能数量。一个做得好的功能比十个做得差的有价值。

## 赢/输分析

### 赢单分析

- 客户选择我们的 Top 3 原因
- 哪些功能/体验是决策关键
- 销售/试用过程中的关键转折点

### 输单分析

- 客户选择竞品的 Top 3 原因
- 我们在哪些维度被明确否定
- 是否有价格/品牌/渠道因素

### 分析模板

```
## [客户名] — [赢/输]

**选择了：** [我们/竞品名]
**决策原因：**
1. [原因 1]
2. [原因 2]
3. [原因 3]

**决策过程：**
- 评估了哪些方案
- 关键决策人是谁
- 评估周期多长

**启示：** [我们能从中学到什么]
```

## 市场趋势分析

**关注 5 个信号：**

1. **融资动态**：竞品最近融了多少钱？投向了什么方向？
2. **招聘方向**：竞品在招什么岗位？（暗示未来发力点）
3. **产品更新**：竞品最近发布了什么功能？节奏如何？
4. **用户评价**：应用商店评分趋势、社交媒体口碑
5. **政策/行业变化**：是否有监管变化影响竞争格局

## 战斗卡模板

给销售/市场团队使用的竞品速查卡：

```
## [竞品名] 战斗卡

**一句话：** [竞品]做得好的是 [X]，但在 [Y] 上不如我们。

**他们的优势：**
- [优势 1]（我们的应对话术：...）
- [优势 2]（我们的应对话术：...）

**他们的劣势：**
- [劣势 1]（我们的对标优势：...）
- [劣势 2]

**客户常问的问题：**
- Q: [问题]
  A: [回答]

**最新动态：** [最近 3 个月的重要变化]

**更新日期：** [日期]
```

## 输出要求

- 数据要标注来源和时间
- 区分"已验证的事实"和"推测"
- 功能对比基于实际体验，不是官网宣传
- 定期更新（建议每季度）
