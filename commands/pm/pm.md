# 产品经理工具箱

为产品经理提供 6 个核心工作流的 AI 辅助。基于 Anthropic 官方 PM 框架适配。

## 启动流程

### 有参数时：自动识别意图

根据 `$ARGUMENTS` 中的关键词匹配功能：

- 含"PRD""需求""功能""spec" → 写 PRD
- 含"用研""访谈""调研""反馈" → 用研分析
- 含"周报""汇报""update""进展" → 写周报/汇报
- 含"路线图""排期""优先级""roadmap" → 路线图排序
- 含"竞品""竞争""对比""分析" → 竞品分析
- 含"指标""数据""复盘""metric" → 数据复盘
- 无法识别 → 弹出选择菜单

### 无参数时：弹出选择菜单

使用 AskUserQuestion 提供 4 个选项（受限于选项数上限）：

**第一组选择：**

| 选项        | 说明                                     |
| ----------- | ---------------------------------------- |
| 写 PRD      | 从问题/想法出发，生成结构化产品需求文档  |
| 用研分析    | 从访谈/问卷/反馈中提炼洞察和机会点       |
| 写周报/汇报 | 按受众生成进展汇报（周报/月报/发布公告） |
| 更多功能... | 路线图排序、竞品分析、数据复盘           |

如果用户选"更多功能"，弹出第二组：

| 选项       | 说明                              |
| ---------- | --------------------------------- |
| 路线图排序 | 用 RICE/MoSCoW 框架给需求排优先级 |
| 竞品分析   | 生成竞品对比矩阵和定位分析        |
| 数据复盘   | 梳理指标体系，分析趋势和异常      |

## 功能 1：写 PRD

**读取** `feature-spec.md` 获取完整框架。

**收集信息（AskUserQuestion）：**

1. 这个功能要解决什么问题？（开放输入）
2. 目标用户是谁？（选择：C 端用户 / B 端客户 / 内部团队 / 其他）
3. 期望什么级别的 PRD？（选择：简版概述 / 标准 PRD / 详细规格）

**输出结构：**

1. 问题陈述（背景 + 痛点 + 机会）
2. 目标与非目标
3. 用户故事（INVEST 原则）
4. 需求分级（MoSCoW：必须/应该/可以/不做）
5. 验收标准（Given/When/Then 格式）
6. 成功指标（先行指标 + 滞后指标）
7. 开放问题
8. 时间线建议

**输出后追问：** "需要我细化哪个部分？还是换个角度重新分析？"

---

## 功能 2：用研分析

**读取** `user-research.md` 获取完整框架。

**收集信息：**

1. 请粘贴或描述你的用研数据（访谈记录/问卷结果/用户反馈/工单）
2. 这次研究想回答什么问题？（开放输入）
3. 数据来源类型？（选择：用户访谈 / 问卷调查 / 客服工单 / 应用评价 / 混合来源）

**输出结构：**

1. 摘要（一段话概括核心发现）
2. 关键主题（每个主题：发现 + 证据引用 + 频次 + 影响 + 置信度）
3. 洞察→机会矩阵（表格：洞察 | 机会 | 影响 | 可行性）
4. 用户分群（如果数据支持）
5. 建议行动项（按优先级排序）
6. 后续研究问题

---

## 功能 3：写周报/汇报

**读取** `stakeholder-comms.md` 获取完整框架。

**收集信息：**

1. 汇报类型？（选择：周报 / 月报 / 发布公告 / 临时通报）
2. 受众是谁？（选择：管理层 / 技术团队 / 跨部门 / 客户）
3. 请提供本周/本月的关键进展、遇到的问题、下一步计划

**输出按受众适配：**

- 管理层：300 字以内，结论先行，G/Y/R 状态灯
- 技术团队：含技术细节，可附代码/架构变更
- 跨部门：聚焦影响面和协作需求
- 客户：正式语气，聚焦价值交付

---

## 功能 4：路线图排序

**读取** `roadmap.md` 获取完整框架。

**收集信息：**

1. 请列出待排序的需求/功能（逐行列出即可）
2. 用哪个框架排序？（选择：RICE 评分 / MoSCoW 分类 / ICE 评分 / 价值-工作量矩阵）
3. 团队规模和迭代周期？（开放输入）

**输出：** 排序后的优先级表格 + 依赖关系图 + 容量建议 + 路线图视图（Now/Next/Later）

---

## 功能 5：竞品分析

**读取** `competitive-analysis.md` 获取完整框架。

**收集信息：**

1. 你的产品是什么？（简述）
2. 要分析哪些竞品？（列出名称，或让 AI 帮你找）
3. 重点关注什么？（选择：功能对比 / 定价策略 / 用户体验 / 市场定位）

**输出结构：**

1. 竞争格局概览
2. 功能对比矩阵（表格，带评分）
3. 定位分析
4. 各竞品的优势/劣势
5. 机会和威胁
6. 建议的差异化策略

---

## 功能 6：数据复盘

**不需要额外 reference 文件，框架内置。**

**收集信息：**

1. 请粘贴或描述你的关键指标数据
2. 复盘周期？（选择：周 / 月 / 季度 / 发布后）
3. 关注哪类指标？（选择：增长指标 / 留存指标 / 收入指标 / 产品健康度 / 全面复盘）

**输出结构：**

1. 指标仪表盘（表格：指标 | 目标 | 实际 | 达成率 | 趋势）
2. 关键变化分析（上升/下降原因推测）
3. 异常信号标记
4. 下一步行动建议

---

## 通用规则

- 所有输出使用中文
- 表格用 Markdown 格式
- 关键结论加粗
- 每个输出最后提供"追问建议"，引导用户深入
- 如果用户提供的信息不足，主动追问而非猜测
- 框架是参考，不是模板——根据实际情况灵活调整，不要机械填充
