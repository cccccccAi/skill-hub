## 第1关：一键出图 vs 一段话

### 验证题

Q: 提升 AI 设计输出质量的最直接方式是什么？

- A. 换一个更贵的 AI 模型
- B. 多运行几次，选最好的
- C. 写更精准的需求描述（产品类型+行业+风格+受众）✓
- D. 提前准备好设计素材

答对：「没错！需求描述的质量直接决定了 AI 输出的质量。」
答错：「正确答案是 C。AI 的输出质量 90% 取决于你的需求描述——产品类型、行业、风格关键词、目标受众。」

### 学习要点

- ✅ 需求描述 4 要素：产品类型 + 行业 + 风格关键词 + 目标受众
- ✅ 同一个工具，需求不同，输出天差地别
- ✅ 第一步就是"想清楚要什么"，这本身就是设计工作

### 原理图

```
┌──────────────────────────────────────────┐
│          需求描述质量 vs 输出质量          │
├──────────────────────────────────────────┤
│                                          │
│  "做个网站"                               │
│       ↓                                  │
│  [BM25 搜索] → 模糊匹配 → 通用结果 😐    │
│                                          │
│  "coffee brand warm artisan cozy"        │
│       ↓                                  │
│  [BM25 搜索] → 精准匹配 → 定制结果 😊    │
│                                          │
│  关键差异：4 个要素的有无                  │
│  产品类型 │ 行业场景 │ 风格词 │ 目标受众   │
│                                          │
└──────────────────────────────────────────┘
```

## 第2关：看懂设计系统

### 验证题

Q: 设计系统中优先级最高（CRITICAL）的两个类别是什么？

- A. 字体和配色
- B. 无障碍（Accessibility）和触摸交互（Touch & Interaction）✓
- C. 性能和响应式布局
- D. 动效和风格选择

答对：「对！无障碍和触摸交互是 CRITICAL 级别——这两个做不好，用户根本无法正常使用。」
答错：「正确答案是 B。无障碍（对比度、alt text）和触摸交互（44×44px 最小目标）是最高优先级，因为它们直接影响用户能否正常使用产品。」

### 学习要点

- ✅ 设计系统 7 大模块：Pattern / Style / Colors / Typography / Effects / Avoid / Checklist
- ✅ 优先级体系：CRITICAL > HIGH > MEDIUM > LOW
- ✅ 看懂输出 → 才能判断哪里需要改 → 这就是设计师的价值

### 原理图

```
┌──────────────────────────────────────────┐
│          设计系统 7 大模块                 │
├──────────────────────────────────────────┤
│                                          │
│  ┌─────────┐  决定页面结构               │
│  │ PATTERN │──────────────────┐          │
│  └─────────┘                  │          │
│  ┌─────────┐  决定视觉感受    │          │
│  │  STYLE  │──────────┐      │          │
│  └─────────┘          ↓      ↓          │
│  ┌─────────┐       ┌──────────────┐     │
│  │ COLORS  │──────→│   最终页面    │     │
│  └─────────┘       └──────────────┘     │
│  ┌─────────┐          ↑      ↑          │
│  │  TYPO   │──────────┘      │          │
│  └─────────┘                  │          │
│  ┌─────────┐  交互层          │          │
│  │ EFFECTS │──────────────────┘          │
│  └─────────┘                             │
│  ┌─────────┐  ┌───────────┐              │
│  │  AVOID  │  │ CHECKLIST │  质量保障层   │
│  └─────────┘  └───────────┘              │
│                                          │
└──────────────────────────────────────────┘
```

## 第3关：配色与字体的人工选择

### 验证题

Q: AI 推荐配色的依据和设计师选配色的依据有什么不同？

- A. AI 看数据，设计师看心情
- B. AI 按产品类型统计匹配，设计师按品牌感觉和目标受众选择 ✓
- C. 没有区别，都是选好看的
- D. AI 更准确，设计师只是确认

答对：「正确！AI 擅长统计匹配（'咖啡品牌通常用棕色'），但设计师知道'我们的品牌走北欧极简风，要冷色调'。」
答错：「正确答案是 B。AI 按产品类型做统计匹配，设计师则基于品牌定位、目标受众的情感需求来选择。两者互补。」

### 学习要点

- ✅ AI 按数据推荐（统计规律），人按品牌选择（情感+上下文）
- ✅ 配色影响品牌调性，字体决定品牌"说话的语气"
- ✅ 人参与选择后，结果从"通用"变成"品牌专属"

### 原理图

```
┌──────────────────────────────────────────┐
│        AI 推荐 vs 人工选择               │
├──────────────────────────────────────────┤
│                                          │
│  AI 推荐路径：                            │
│  "咖啡品牌" → 数据库匹配 → 棕色系 #8B4513│
│  依据：96 个配色方案的产品类型标签         │
│                                          │
│  设计师选择路径：                          │
│  "我们走北欧简约" → 品牌定位 → 冷灰系     │
│  依据：品牌策略 + 目标受众 + 情感意图      │
│                                          │
│  ──────────────────────────────           │
│  结论：AI 提供选项池，设计师做最终判断     │
│                                          │
└──────────────────────────────────────────┘
```

## 第4关：设计评审找问题

### 验证题

Q: 以下哪个设计问题的严重程度是 CRITICAL？

- A. 用 emoji 做图标
- B. 动画超过 300ms
- C. 文字对比度低于 4.5:1 ✓
- D. 按钮没有 hover 效果

答对：「对！对比度不足（< 4.5:1）是 CRITICAL 级别——视力不好的用户完全看不清内容。」
答错：「正确答案是 C。文字对比度是无障碍的核心要求（WCAG AA 标准），属于 CRITICAL 级别。emoji 图标和 hover 效果是 MEDIUM，动画时长也是 MEDIUM。」

### 学习要点

- ✅ 设计系统是"审查"工具，不只是"生成"工具
- ✅ 评审优先级：CRITICAL（对比度、触摸） > HIGH（响应式） > MEDIUM（图标、hover）
- ✅ 专业设计师的价值在于发现 AI 输出中的问题

### 原理图

```
┌──────────────────────────────────────────┐
│          设计评审流程                      │
├──────────────────────────────────────────┤
│                                          │
│  AI 生成页面                              │
│       ↓                                  │
│  ┌─────────────┐                         │
│  │ 人工评审     │ ← 用 8 个优先级规则检查  │
│  └─────────────┘                         │
│       ↓                                  │
│  发现问题清单                             │
│  ├── CRITICAL: 对比度、触摸目标           │
│  ├── HIGH: 响应式、性能                   │
│  └── MEDIUM: 图标、hover、动画            │
│       ↓                                  │
│  逐个修复（CRITICAL 优先）                │
│       ↓                                  │
│  专业级输出                               │
│                                          │
└──────────────────────────────────────────┘
```

## 第5关：粗稿 vs 精稿

### 验证题

Q: 设计系统的 Master + Overrides 架构有什么好处？

- A. 可以给不同客户用不同设计
- B. 全局一致性 + 页面级灵活性 ✓
- C. 减少文件数量
- D. 方便 AI 自动生成

答对：「对！Master 保证全局一致，Overrides 允许特定页面有自己的个性。比如首页可以更活泼，产品页更专业。」
答错：「正确答案是 B。Master 文件定义全局规则确保一致性，页面级覆盖文件允许特定页面有差异化需求，两者结合。」

### 学习要点

- ✅ "粗稿→精稿"的过程就是人的判断不断叠加的过程
- ✅ 每个环节的人工介入都有具体价值：需求/配色/字体/评审/持久化
- ✅ AI 是画笔，设计师是画家——画笔越好，画家的判断就越重要

### 原理图

```
┌──────────────────────────────────────────┐
│        粗稿 → 精稿：5 层人工判断          │
├──────────────────────────────────────────┤
│                                          │
│  Layer 0: AI 全自动                       │
│  ════════════════════                    │
│       ↓  + 精准需求描述                   │
│  Layer 1: 需求层（第1关）                 │
│  ════════════════════                    │
│       ↓  + 理解输出、发现问题              │
│  Layer 2: 理解层（第2关）                 │
│  ════════════════════                    │
│       ↓  + 人工选配色/字体/风格           │
│  Layer 3: 选择层（第3关）                 │
│  ════════════════════                    │
│       ↓  + 设计评审修复问题               │
│  Layer 4: 评审层（第4关）                 │
│  ════════════════════                    │
│       ↓  + 持久化 + 页面覆盖             │
│  Layer 5: 精稿（第5关）                   │
│  ════════════════════                    │
│                                          │
│  每一层 = 一次人的判断                    │
│  5层叠加 = 从"能用"到"好用"              │
│                                          │
└──────────────────────────────────────────┘
```
