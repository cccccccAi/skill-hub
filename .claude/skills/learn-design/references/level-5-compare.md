# 第5关：粗稿 vs 精稿

## 目标

完整走一遍精细化流程，最终对比第 1 关的粗稿。这是整个教学的高潮——用户亲眼看到"多步人工介入"的累积效果。

## 教学流程

### Step 1 — 回顾全程进展

读取 `./design/output/learn-progress.md`，简要回顾（不超过 6 行）：

```
第1关 → 粗稿 v0（"做个网站"→ AI 全自动）
第1关 → v0.5（精准需求 → AI 全自动选择）
第3关 → 精稿 v1（+ 人工选配色/字体/风格）
第4关 → v1.5（+ 人工评审发现问题）
本关  → 最终版（+ 持久化 + 页面覆盖 + 生成成品）
```

### Step 2 — 持久化设计系统

讲解 Master + Overrides 架构（不超过 6 行）：

> 真实项目中，设计系统需要"保存"下来反复使用。
> Master 文件是全局规则，每个页面可以有自己的覆盖。
> 比如：首页要更活泼的配色，产品页要更专业的风格。

用户执行持久化命令：

```bash
python3 ./design/engine/scripts/search.py "<精炼需求关键词>" --design-system --persist -p "<项目名>" --page "homepage" -o ./design/output
```

检查输出：
- `./design/output/<项目 slug>/MASTER.md` 应存在
- `./design/output/<项目 slug>/pages/homepage.md` 应存在

### Step 3 — 生成最终 HTML 页面

基于精稿 v1.5 的设计系统（用户选的配色 + 字体 + 风格 + 评审修复），AI 生成一个完整的 HTML 页面。

**生成规则**：
- 使用用户在第 3 关选择的配色方案
- 使用用户在第 3 关选择的字体（含 Google Fonts import）
- 应用第 4 关评审的所有修复（cursor-pointer、对比度、touch target 等）
- 包含响应式布局（375px / 768px / 1024px）
- 使用 SVG 图标而非 emoji
- 所有动画 150-300ms

保存到 `./design/output/<项目 slug>/homepage.html`

展示时告诉用户：
> 生成了最终版页面，融合了你在 4 关中做的所有决策。

### Step 5 — 总结金句

> **AI 是画笔，设计师是画家。画笔越好，画家的判断就越重要——不是更不重要。**

用 1-2 段话总结教学核心（不超过 6 行）：
- 不是 AI 不够好，而是好结果需要好判断
- 需求描述、配色选择、字体搭配、设计评审——每个环节都是人的价值
- 工具只会越来越强，但"这个品牌应该给人什么感觉"永远是设计师的工作
