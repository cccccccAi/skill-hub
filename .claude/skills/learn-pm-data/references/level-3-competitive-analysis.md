# 第3关：竞品分析

> 注意：如果行业包配置了竞品排除规则（如母公司关系），需遵守排除条件。具体规则见 pack-config.md「竞品规则」。

## 目标

用 competitive-analysis 框架产出一份竞品简报，包含竞品地图、功能矩阵、定位分析和战略建议。

## 时长

约 12 分钟

## 教学流程

### Step 1：场景设定

告诉用户：「当前使用 [数据来源] 路线。随时可以补充真实竞品情报，让分析更有针对性。这一关做竞品分析。」

### Step 2：选择竞品

告诉用户：「领导让你做一份竞品分析。先确定分析对象。」

用 AskUserQuestion 问：「选择 2 个你最想分析的竞品：」（多选）
选项从 pack-config.md「Level 3 配置 > 竞品列表」读取。

**真实数据路线**：用户可以补充自己掌握的竞品信息（产品动态、客户反馈、丢单原因等），AI 会整合到分析中。

### Step 3：竞品地图

读取 `./references/sample-data/{pack}/competitive.md` 获取预置竞品数据。如果用户提供了额外信息，整合到分析中。

展示竞品分类地图（来自 competitive-analysis skill），竞品分类从竞品数据文件和 pack-config.md「Level 3 配置 > 竞品分类」读取：

- **直接竞品**：同样的问题、同样的用户、同样的方式
- **相邻竞品**：目前不直接竞争但有可能进入
- **替代方案**：完全不同的解决方式

关键教学点：「最危险的竞品不一定是现在和你直接抢客户的。注意那些从其他领域跨界进入的玩家——这种降维打击往往比正面竞争更难防守。」

### Step 4：功能对比维度

用 AskUserQuestion 问：「功能对比矩阵需要选维度。以下哪 3 个维度最影响客户采购决策？」（多选）
选项从 pack-config.md「Level 3 配置 > 功能对比维度」读取。

### Step 5：生成功能矩阵

AI 基于用户选择的维度和竞品，生成功能对比矩阵：

```
| 能力维度 | {产品名称} | [竞品1] | [竞品2] |
|---------|-----------|---------|---------|
| [维度1] | ★★★★☆    | ★★★☆☆  | ★★☆☆☆  |
| [维度2] | ★★★☆☆    | ★★★★☆  | ★★★☆☆  |
| [维度3] | ★★★★★    | ★★☆☆☆  | ★★★☆☆  |
```

关键教学点：「功能矩阵要诚实。一个到处都赢的矩阵没有可信度。承认对手在某些维度的优势，才能让你在其他维度的优势更有说服力。」

### Step 6：定位分析

AI 展示每个竞品的定位声明（Positioning Statement）：

```
对于 [目标客户] 中有 [需求] 的人，
[产品] 是一个 [品类]，
它的核心价值是 [差异化优势]。
不同于 [竞品]，[产品] 的独特之处在于 [关键差异]。
```

用 AskUserQuestion 问：「基于分析，{产品名称}最强的定位方向是？」
选项从 pack-config.md「Level 3 配置 > 定位方向选项」读取。

### Step 7：战略建议

AI 基于全部分析生成 3-5 条战略建议（Strategic Implications），每条包含：

- **发现**：基于竞品分析的关键洞察
- **建议**：具体的产品策略动作
- **优先级**：高/中/低

基于竞品数据包中各竞品的优劣势和 pack-config.md「Level 3 配置 > 关键竞品洞察」，生成 3-5 条战略建议。

### Step 8：实战提示卡

展示实战提示卡：

```
以后做竞品分析时，对 AI 说：
"帮我做一份竞品分析简报。
我的产品：[产品名]，[一句话定位]
目标竞品：[竞品名1]、[竞品名2]
重点分析维度：[维度1]、[维度2]、[维度3]
我知道的竞品信息：[粘贴你掌握的情报]
请按 competitive-brief 框架输出：
竞品地图 → 功能矩阵 → 定位分析 → 战略建议"
```

### Step 9：生成产出

将完整竞品简报写入 `./learning/outputs/level-3-competitive.md`，格式：

```markdown
# {产品名称}竞品分析简报

## 分析范围

- 分析对象：[用户选择的竞品]
- 分析维度：[用户选择的维度]
- 分析时间：[当前时间]

## 竞品地图

[分类说明]

## 功能对比矩阵

[评分表格]

## 定位分析

### {产品名称}

[定位声明]

### [竞品1]

[定位声明]

### [竞品2]

[定位声明]

## 优势与劣势

### 我们的优势

[2-3 条]

### 我们的劣势

[2-3 条，诚实]

## 战略建议

[3-5 条，按优先级排序]

## 监控计划

[需要持续跟踪的竞品动态]

## 实战提示卡

[prompt 模板]
```

## 验证条件

- 用户完成了竞品选择和功能维度选择
- 用户完成了定位方向判断
- `./learning/outputs/level-3-competitive.md` 文件已生成
