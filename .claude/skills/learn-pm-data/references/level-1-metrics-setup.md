# 第1关：指标体系整理

> 合并了插件快速热身 + 指标体系搭建。产品总监不需要花 10 分钟学架构，3 分钟搞懂机制就够了。

## 目标

3 分钟理解插件机制 + 把现有零散指标整理成 North Star → L1 → L2 体系。

## 时长

约 15 分钟

## 教学流程

### Step 1：插件热身（3 分钟）

告诉用户：「在正式开始前，花 3 分钟理解训练营背后的机制。整个训练营基于 Anthropic 的 knowledge-work-plugins，它的核心结构就两样东西：」

展示一个简表：

| 概念            | 类比   | 作用                                               |
| --------------- | ------ | -------------------------------------------------- |
| Skill（技能）   | 教科书 | 给 AI 的知识框架（如：什么是 North Star 指标）     |
| Command（命令） | 工作流 | 触发一个分步任务（如：/metrics-review 做指标复盘） |

「接下来 6 关，每一关对应的 skill 和 command 是：」

| 关卡           | 使用的 Skill                  | 使用的 Command       |
| -------------- | ----------------------------- | -------------------- |
| 第1关 指标整理 | metrics-tracking              | /metrics-review      |
| 第2关 用研综合 | user-research-synthesis       | /synthesize-research |
| 第3关 竞品分析 | competitive-analysis          | /competitive-brief   |
| 第4关 数据叙事 | stakeholder-comms             | /stakeholder-update  |
| 第5关 数据看板 | interactive-dashboard-builder | /build-dashboard     |
| 第6关 创建技能 | data-context-extractor        | —                    |

「热身结束。记住一句话就够：**Skill 是 AI 的知识库，Command 是你触发的工作流**。后面每一关你会亲手体验。」

### Step 2：进入正题

告诉用户：「当前使用 [数据来源] 路线。随时可以补充真实数据。现在进入正题——指标体系整理。」

介绍三层指标体系（来自 metrics-tracking skill）：

- **North Star（北极星指标）**：一个指标，代表产品交付的核心价值
- **L1 健康指标（5-7个）**：获客、激活、留存、营收、满意度
- **L2 诊断指标**：当 L1 出问题时，用来定位原因

关键教学点：「North Star 不是'收入'或'用户数'——它是你的产品交付核心价值的度量。」

### Step 3A：真实数据路线 — 整理指标

如果用户选择真实数据路线：

1. 请用户粘贴或描述他们目前在看的指标（格式不限，可以是列表、表格、口头描述）
2. AI 将用户提供的指标归类到 North Star / L1 / L2 框架中：
   - 识别哪个最适合做 North Star
   - 将其余指标归入 L1 的获客/激活/留存/营收/满意度维度
   - 识别哪些是 L2 诊断指标
   - **标注缺失维度**：如果某个 L1 维度没有对应指标，提醒用户
3. 展示整理后的框架，用 AskUserQuestion 问：「这个归类你觉得合理吗？」
   - "合理，继续"
   - "我想调整 North Star 的选择"
   - "某个指标归错维度了，我来说明"

### Step 3B：行业数据包路线 — 选择指标

如果用户选择了行业数据包：

用 AskUserQuestion 问：「对{产品名称}来说，最能代表核心价值的 North Star 是？」
选项从当前行业包的 pack-config.md「Level 1 配置 > North Star 候选」部分读取。

无论用户选什么，给出正面反馈并解释为什么这是个好选择。

逐个维度通过 AskUserQuestion 让用户选择 L1 指标，选项从 pack-config.md「Level 1 配置 > L1 维度选项」各维度读取。

根据用户选择，AI 生成完整的 L1/L2 指标框架（包含自动推导的 L2 诊断指标）。

### Step 4：设定一个 OKR

用 AskUserQuestion 问：「下个季度最想改善的指标维度是？」
选项从 pack-config.md「Level 1 配置 > OKR 维度选项」读取。真实数据路线的用户基于自己的指标选择。

基于选择，AI 展示一个 OKR 范例（Objective + 3 Key Results），并让用户确认或调整。

### Step 5：模拟指标复盘

**真实数据路线**：如果用户在 Step 3A 提供了真实数据，用真实数据做复盘演示。

**行业数据包路线**：读取预置数据 `./references/sample-data/{pack}/metrics.md`。

告诉用户：「现在用你刚建立的框架，做一次指标复盘。」

按照 metrics-review 的工作流执行：

1. 展示指标记分卡（Scorecard 表格）
2. 标出趋势（特别关注异常数据点）
3. 分析可能原因
4. 提出 2-3 个建议动作

用 AskUserQuestion 追问，提问内容和选项从 pack-config.md「Level 1 配置 > 指标复盘追问」读取。

如果是真实数据路线，针对用户数据中的异常点设计类似的追问。

### Step 6：实战提示卡

告诉用户：「这一关你学到了指标层级框架和复盘方法。送你一张实战提示卡——以后不需要教程，直接用这个 prompt 模板：」

展示实战提示卡：

```
以后做指标复盘时，直接对 AI 说：
"帮我做一次产品指标复盘。我的 North Star 是 [XXX]，
L1 指标包括 [粘贴指标列表]。
以下是最近的数据 [粘贴数据表格]。
请按 metrics-review 的框架分析趋势、标出异常、给出建议动作。"
```

### Step 7：生成产出

将完整的指标体系 + OKR + 复盘报告 + 实战提示卡写入 `./learning/outputs/level-1-metrics.md`，格式：

```markdown
# {产品名称}指标体系

## North Star

[指标名] — [一句话解释]

## L1 健康指标

| 维度   | 指标 | 定义 | 当前基准 |
| ------ | ---- | ---- | -------- |
| 获客   | ...  | ...  | ...      |
| 激活   | ...  | ...  | ...      |
| 留存   | ...  | ...  | ...      |
| 营收   | ...  | ...  | ...      |
| 满意度 | ...  | ...  | ...      |

## L2 诊断指标

[按维度列出]

## OKR（下季度）

**Objective**: ...

- KR1: ...
- KR2: ...
- KR3: ...

## 指标复盘记录

[复盘内容]

## 实战提示卡

[prompt 模板]
```

## 验证条件

- 用户完成了指标框架搭建（North Star + L1）
- 用户完成了 OKR 设定
- `./learning/outputs/level-1-metrics.md` 文件已生成
