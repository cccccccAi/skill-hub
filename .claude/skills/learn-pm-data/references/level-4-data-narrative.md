# 第4关：数据叙事

> 这是训练营中用户最迫切需要的能力——用数据说服不同受众。

## 目标

学会"用数据讲故事"——不是把数据摆出来，而是让数据说服人。掌握"现状→洞察→行动"三段式和受众适配技巧。

## 路线提示

> 当前路线（真实数据 / 模拟数据）在启动时已确定，本关不再重复选择。

## 时长

约 12 分钟

## 教学流程

### Step 1：为什么需要数据叙事

告诉用户：「你手上有大量数据。但数据本身不会说话——你需要把数据变成一个能说服人的故事。这就是数据叙事（Data Narrative）。」

关键教学点：「数据展示墙 ≠ 数据叙事。展示墙是"把数据摆出来让你自己看"，叙事是"我用数据告诉你该做什么决策"。」

介绍数据叙事三段式（来自 stakeholder-comms skill）：

| 段落                  | 作用                     | 时间                 |
| --------------------- | ------------------------ | -------------------- |
| **现状（Situation）** | 关键数据快照             | 受众 10 秒内看完     |
| **洞察（Insight）**   | 数据说明了什么问题或机会 | 30 秒理解核心发现    |
| **行动（Action）**    | 基于洞察的建议下一步     | 1 分钟内知道该做什么 |

### Step 2：受众适配方法论

告诉用户：「同样的数据，给不同的人看，故事完全不同。」

展示受众适配原则（具体受众从 pack-config.md「Level 4 配置 > 受众适配规则」读取）：

| 受众类型          | 关心什么                             | 数据密度             | 语言风格                        |
| ----------------- | ------------------------------------ | -------------------- | ------------------------------- |
| **高层决策者**    | 宏观覆盖、战略对齐、整体效果         | 低（3-5 个核心数字） | 结论先行，用百分比和排名        |
| **中层管理者**    | 本单位 vs 整体、改进证据、投入产出比 | 中（单位级对比表）   | 对比驱动，"我们在整体中排名..." |
| **执行/产品团队** | 功能数据、使用漏斗、留存、异常       | 高（详细图表+下钻）  | 数据驱动，关注趋势和归因        |

关键教学点：

- 给高层：**结论先行**，"我们覆盖了 X 个客户，核心指标平均提升 Y%"
- 给中层：**对比驱动**，"本单位核心指标提升 Z，高于整体平均水平"
- 给团队：**归因为主**，"功能 A 使用者 D30 留存 85%，未使用者 51%"

### Step 3：状态信号

介绍 Green/Yellow/Red 状态信号（来自 stakeholder-comms skill）：

| 颜色      | 含义 | 何时使用               |
| --------- | ---- | ---------------------- |
| 🟢 Green  | 正轨 | 指标达标或超出预期     |
| 🟡 Yellow | 风险 | 指标偏离但有缓解计划   |
| 🔴 Red    | 偏离 | 显著落后，需要升级处理 |

关键教学点：「Yellow 不是失败，是诚实的风险管理。最糟糕的数据叙事是把 Yellow 伪装成 Green——领导发现时信任就崩了。」

### Step 4：受众选择

用 AskUserQuestion 问：「你最近需要给谁用数据汇报？」
选项从 pack-config.md「Level 4 配置 > 受众选项」读取。

> 真实数据路线的用户基于自己的受众列表选择。

### Step 5：生成数据叙事

AI 根据用户选择的受众和数据，生成一份完整的数据叙事文档。

**数据来源**：

- 模拟数据路线：从行业包的 metrics 数据读取，或使用预置样本数据
- 真实数据路线：使用用户提供的真实数据

**文档结构（以高层决策者版为例）**：

```markdown
# {产品名称}数据价值报告

致：[受众角色]

## 现状快照（10 秒版）

| 指标        | 数值   | 变化   | 状态 |
| ----------- | ------ | ------ | ---- |
| [核心指标1] | [数值] | [变化] | 🟢   |
| [核心指标2] | [数值] | [变化] | 🟢   |
| [核心指标3] | [数值] | [变化] | 🟢   |
| [核心指标4] | [数值] | [变化] | 🟡   |

## 核心洞察

### 洞察1：[用数据得出的结论性标题]

[数据 + 解释 + 对比]

### 洞察2：[某维度表现最突出]

[数据 + 归因 + 建议]

### 洞察3：[某方面存在风险（需关注）]

[数据 + 风险 + 缓解计划]

## 建议行动

1. [具体建议 + 预期效果]
2. [具体建议 + 预期效果]
3. [具体建议 + 预期效果]
```

**文档原则**（确保 AI 遵循）：

- 标题写洞察不写数据名（"数学进步最大"而非"各科平均分"）
- 每个洞察都有"so what"——不只是报数字，而是说明意义
- 行动建议具体可执行，不是"继续观察"
- 状态颜色诚实：Yellow 就是 Yellow

### Step 6：审查与修改

展示生成的叙事文档核心部分，用 AskUserQuestion 问：「你觉得需要调整什么？」

- "语气/措辞需要调整"
- "想换个受众版本看看"
- "数据重点要调整"
- "很好，不需要改"

如果需要修改，执行一次修改。如果想换受众版本，基于同样的数据生成另一个版本供对比。

### Step 7：实战提示卡

展示实战提示卡：

```
以后需要用数据汇报时，对 AI 说：
"帮我基于以下数据，写一份给 [受众] 的数据汇报。
受众关心的是：[他们的决策场景]
数据：[粘贴数据]
请用'现状→洞察→行动'三段式结构，
控制在 [1页/2页] 以内，
标题要写洞察而非数据名。"
```

### Step 8：生成产出

将数据叙事文档 + 实战提示卡写入 `./learning/outputs/level-4-data-narrative.md`，格式：

```markdown
# 数据叙事文档

## 元信息

- 目标受众：[受众角色]
- 数据来源：[真实数据/模拟数据]
- 生成时间：[日期]

## 叙事正文

[完整的现状→洞察→行动文档]

## 方法论备忘

- 三段式：现状（10秒）→ 洞察（30秒）→ 行动（1分钟）
- 受众适配：密度、语言、关注点都要匹配受众
- 状态信号：Green/Yellow/Red 诚实标注
- 标题原则：写洞察不写数据名

## 实战提示卡

[prompt 模板]
```
