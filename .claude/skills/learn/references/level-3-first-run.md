# 第3关：运行第一个 Skill（实战）

## 目标

让用户亲手运行一个 Skill，理解完整的执行流程。

## 教学流程

### 1. 选择主题

使用 AskUserQuestion 提供 4 个可选主题：

```
你想聊什么话题？选一个最感兴趣的：

1. 转行/跨界做新领域的心路历程
2. 第一次用 AI 工具完成工作任务
3. 一次跨部门协作中的意外收获
4. AI 正在改变我的工作方式
```

记录用户选择的主题，后续传给 `/interview-write`。

### 2. 工作空间说明

> 接下来 Skill 会开始运行，先告诉你文件会保存在哪：
>
> ```
> 📂 文件位置：
>   访谈记录 → writing/drafts/interview-transcript.md
>   最终文章 → writing/output/日期-标题.md
> ```
>
> 你不需要手动操作这些文件，Skill 会自动处理。

### 3. 记录执行前状态

先记录当前 `./writing/output/` 下的文件数量：

```bash
ls ./writing/output/ | wc -l
```

保存这个数字，后面验证用。

### 4. 引导运行

> 现在来实战！我们要运行 interview-write 这个 Skill。
>
> 请退出当前的 learn 教学（输入 `/exit`），然后运行：
>
> ```
> /interview-write {用户选择的主题}
> ```
>
> 这个 Skill 会以访谈的方式和你聊天，挖掘你的故事，最终写出一篇文章。
>
> 整个过程大概需要几分钟。完成后，再回来运行 `/learn 3` 继续闯关。
>
> **提示：** 访谈记录会自动保存，第4关会复用这份素材，不用重新聊一遍。

### 5. 用户回来后

用户运行 `/learn 3` 回来时，检查 `./writing/output/` 下是否有新文件。

### 6. 复盘 — 终端展示终稿

如果有新文件，**读取文件内容并在终端直接展示全文**：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📄 你的第一篇 Skill 产出
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

{读取并展示文章全文}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📁 已保存到：writing/output/{文件名}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

简要点评：
- 指出文章中的金句（加粗部分）
- 指出风格参考的痕迹
- 说明这篇文章是怎么从访谈内容转化来的

## 验证

1. `./writing/output/` 下有新的 .md 文件（文件数比执行前多）
2. 新文件字数 > 200 字符

通过后显示：

```
✅ 第3关通过！你已经成功运行了第一个 Skill，并产出了一篇文章。
```

然后执行通关总结（见 summaries.md）。

## 未通过处理

如果用户回来但没有新文件：

> 看起来文章还没生成。可能的原因：
> 1. Skill 运行过程中中断了
> 2. 访谈还没结束就退出了
>
> 试着重新运行 `/interview-write {之前选的主题}`，完整走完访谈和写作流程后再回来。
