# 第3关：运行第一个 Skill（实战）

## 目标

让用户亲手运行一个 Skill，理解完整的执行流程。

## 教学流程

### 1. 记录执行前状态

先记录当前 `./writing/output/` 下的文件数量：

```bash
ls ./writing/output/ | wc -l
```

保存这个数字，后面验证用。

### 2. 引导运行

> 现在来实战！我们要运行 interview-write 这个 Skill。
>
> 请退出当前的 learn 教学（输入 `/exit`），然后运行：
>
> ```
> /interview-write 春节期间有趣的事情
> ```
>
> 这个 Skill 会以访谈的方式和你聊天，挖掘你的故事，最终写出一篇文章。
>
> 整个过程大概需要几分钟。完成后，再回来运行 `/learn 3` 继续闯关。

### 3. 用户回来后

用户运行 `/learn 3` 回来时，检查 `./writing/output/` 下是否有新文件。

### 4. 复盘

如果有新文件，读取并展示给用户：

> 看看你的第一篇 Skill 产出！

简要点评：
- 指出文章中的金句（加粗部分）
- 指出风格参考的痕迹
- 说明这篇文章是怎么从访谈内容转化来的

## 验证

1. `./writing/output/` 下有新的 .md 文件（文件数比执行前多）
2. 新文件字数 > 200 字符

通过后显示：

```
✅ 第3关通过！你已经成功运行了第一个 Skill，并产出了一篇文章。
```

## 未通过处理

如果用户回来但没有新文件：

> 看起来文章还没生成。可能的原因：
> 1. Skill 运行过程中中断了
> 2. 访谈还没结束就退出了
>
> 试着重新运行 `/interview-write 春节期间有趣的事情`，完整走完访谈和写作流程后再回来。
