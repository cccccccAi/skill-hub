# 第3关：运行第一个 Skill（实战）

## 目标

让用户亲手运行一个 Skill，理解完整的执行流程。

## 教学流程

### 1. 选择主题

使用 AskUserQuestion 提供 4 个可选主题：

```
你想聊什么话题？选一个最感兴趣的：

1. 转行/跨界做新领域的心路历程
2. 第一次用 AI 工具完成工作任务
3. 一次跨部门协作中的意外收获
4. AI 正在改变我的工作方式
```

记录用户选择的主题。

### 2. 工作空间说明

> 接下来我会直接启动访谈写作流程，先告诉你文件会保存在哪：
>
> ```
> 📂 文件位置：
>   访谈记录 → writing/drafts/interview-transcript.md
>   最终文章 → writing/output/日期-标题.md
> ```
>
> 你不需要手动操作这些文件，Skill 会自动处理。
>
> **个性化提示：** `writing/examples/` 里存放了风格参考文章，
> 写作 Skill 会学习这些案例的风格。你可以随时替换成自己喜欢的文章，
> 下次运行时就会自动使用新风格。

### 3. 记录执行前状态

先记录当前 `./writing/output/` 下的文件数量：

```bash
ls ./writing/output/ | wc -l
```

保存这个数字，后面验证用。

### 4. 直接启动访谈写作流程

**不要让用户退出再手动输入命令。** 选完主题后，直接在当前会话中按照 interview-write 的协议执行访谈和写作：

1. 读取 `.claude/skills/interview-write/SKILL.md` 获取完整执行流程
2. 读取 `.claude/skills/interview-write/references/style-guide.md` 获取写作规范
3. 按照 interview-write 的 Phase 1-7 依次执行：
   - Phase 1: 用选定的主题开场
   - Phase 2: 深度访谈（共情追问，实时保存到 drafts/interview-transcript.md）
   - Phase 3: 判断终止
   - Phase 4: 读取 `./writing/examples/` 学习风格
   - Phase 5: 写作输出
   - Phase 6: 自动优化
   - Phase 7: 终端展示

整个访谈和写作在当前会话内完成，用户无需退出。

> **提示：** 访谈记录会自动保存到 drafts/interview-transcript.md，第4关会复用这份素材，不用重新聊一遍。

### 5. 复盘 — 终端展示终稿

写作完成后，**读取文件内容并在终端直接展示全文**：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📄 你的第一篇 Skill 产出
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

{读取并展示文章全文}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📁 已保存到：writing/output/{文件名}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

简要点评：
- 指出文章中的金句（加粗部分）
- 指出风格参考的痕迹
- 说明这篇文章是怎么从访谈内容转化来的

## 验证

1. `./writing/output/` 下有新的 .md 文件（文件数比执行前多）
2. 新文件字数 > 200 字符

通过后显示：

```
✅ 第3关通过！你已经成功运行了第一个 Skill，并产出了一篇文章。
```

然后执行通关总结（见 summaries.md）。

## 未通过处理

如果写作流程中断或产出不完整：

> 看起来文章没有生成完整。我们重新试一次，我会再次启动访谈流程。
