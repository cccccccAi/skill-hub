# 第4关：团队模式体验

## 目标

让用户体验多 Agent 协作写作，理解单人模式和团队模式的区别。

## 教学流程

### 1. 讲解团队架构

> 上一关你用的是单人模式——一个 Agent 完成所有工作。
> 现在我们来试试团队模式，4 个 Agent 分工协作：

```
📋 Team Lead（你）
    ├── 🎤 Interviewer — 负责深度访谈
    ├── ✍️ Writer A（悬念派）— 用悬念和反转吸引读者
    ├── ✍️ Writer B（洞察派）— 用洞察和金句打动读者
    └── 🔍 Critic — 6维度对比评审两篇草稿
```

> 两个写手会根据同样的访谈素材，用不同风格各写一篇。
> 然后 Critic 对比评审，最后 Team Lead 综合两篇的优点写出终稿。

### 2. 记录执行前状态

```bash
ls ./writing/output/ | wc -l
ls ./writing/drafts/ 2>/dev/null | wc -l
```

### 3. 引导运行

> 准备好了吗？退出当前教学（`/exit`），然后运行：
>
> ```
> /interview-write-team 春节期间有趣的事情
> ```
>
> 团队模式比单人模式耗时更长，因为有多个 Agent 在并行工作。
> 完成后运行 `/learn 4` 回来继续。

### 4. 用户回来后

检查产出，然后对比分析：

> 来看看团队模式的成果：

- 展示 drafts 中的两篇草稿（Writer A vs Writer B）
- 展示 Critic 的评审意见（如果有）
- 展示最终输出

> **对比上一关的单人模式：**
> - 单人模式：一个人从头到尾，风格统一
> - 团队模式：多视角碰撞，取各家之长
>
> 哪种你更喜欢？
