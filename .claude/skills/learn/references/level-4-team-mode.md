# 第4关：团队模式体验

## 目标

让用户体验多 Agent 协作写作，理解单人模式和团队模式的区别。复用第3关的访谈素材，省去重复访谈。

## 教学流程

### 1. 检查第3关素材

检查 `./writing/drafts/interview-transcript.md` 是否存在。

- **存在** → 继续教学
- **不存在** → 提示用户：
  > 第4关需要用到第3关的访谈素材，但没找到记录。
  > 请先完成第3关：`/learn 3`

### 2. 讲解团队架构

> 上一关你用的是单人模式——一个 Agent 完成所有工作。
> 现在我们来试试团队模式，4 个 Agent 分工协作：

```
📋 Team Lead（你）
    ├── 🎤 Interviewer — 负责深度访谈
    ├── ✍️ Writer A（悬念派）— 用悬念和反转吸引读者
    ├── ✍️ Writer B（洞察派）— 用洞察和金句打动读者
    └── 🔍 Critic — 6维度对比评审两篇草稿
```

> 两个写手会根据同样的访谈素材，用不同风格各写一篇。
> 然后 Critic 对比评审，最后 Team Lead 综合两篇的优点写出终稿。
>
> **这次我们直接复用上一关的访谈素材**，不用再聊一遍，直接看团队怎么协作。

### 3. 记录执行前状态

```bash
ls ./writing/output/ | wc -l
```

### 4. 直接启动团队写作流程

**不要让用户退出再手动输入命令。** 直接在当前会话中按照 interview-write-team 的协议执行团队写作：

1. 读取 `.claude/skills/interview-write-team/SKILL.md` 获取完整执行流程
2. 因为访谈素材已存在（`./writing/drafts/interview-transcript.md`），跳过访谈阶段
3. 按照 interview-write-team 的 Step 1-7 执行：
   - 创建 Agent Team
   - 跳过访谈（素材已有）
   - 并行写作（Writer A 悬念派 + Writer B 洞察派）
   - Critic 评审
   - 综合终稿
   - 终端展示
   - 清理团队

### 5. 完整展示协作过程

**重点：这一步要让用户看到完整的"对抗→评审→融合"全链路，不能只摘亮点。** 用户需要亲眼看到两篇完全不同风格的文章是怎么通过评审和综合变成更好的终稿的。

| 对比维度 | 单人模式 | 团队模式 | 差异分析 |
|---------|---------|---------|---------|
| 开头 | {引用单人开头} | {引用团队开头} | 团队版融合了悬念+洞察 |
| 核心金句 | {引用} | {引用} | 团队版金句来自两个写手的碰撞 |
