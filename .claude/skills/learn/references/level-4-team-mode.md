# 第4关：团队模式体验

## 目标

让用户体验多 Agent 协作写作，理解单人模式和团队模式的区别。复用第3关的访谈素材，省去重复访谈。

## 教学流程

### 1. 检查第3关素材

检查 `./writing/drafts/interview-transcript.md` 是否存在。

- **存在** → 继续教学
- **不存在** → 提示用户：
  > 第4关需要用到第3关的访谈素材，但没找到记录。
  > 请先完成第3关：`/learn 3`

### 2. 讲解团队架构

> 上一关你用的是单人模式——一个 Agent 完成所有工作。
> 现在我们来试试团队模式，4 个 Agent 分工协作：

```
📋 Team Lead（你）
    ├── 🎤 Interviewer — 负责深度访谈
    ├── ✍️ Writer A（悬念派）— 用悬念和反转吸引读者
    ├── ✍️ Writer B（洞察派）— 用洞察和金句打动读者
    └── 🔍 Critic — 6维度对比评审两篇草稿
```

> 两个写手会根据同样的访谈素材，用不同风格各写一篇。
> 然后 Critic 对比评审，最后 Team Lead 综合两篇的优点写出终稿。
>
> **这次我们直接复用上一关的访谈素材**，不用再聊一遍，直接看团队怎么协作。

### 3. 记录执行前状态

```bash
ls ./writing/output/ | wc -l
```

### 4. 引导运行

> 准备好了吗？退出当前教学（`/exit`），然后运行：
>
> ```
> /interview-write-team {第3关的主题}
> ```
>
> 因为访谈素材已经存在，团队会跳过访谈直接进入对抗写作。
> 完成后运行 `/learn 4` 回来继续。

### 5. 用户回来后 — 展示协作过程

检查产出，然后展示完整的协作过程：

#### 5.1 展示两篇草稿亮点

分别读取 `./writing/drafts/draft-a-suspense.md` 和 `./writing/drafts/draft-b-insight.md`，摘录各自的亮点：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✍️ Writer A（悬念派）亮点：
  {摘录1-2个最精彩的段落或金句}

✍️ Writer B（洞察派）亮点：
  {摘录1-2个最精彩的段落或金句}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 5.2 展示 Critic 评审结论

读取 `./writing/drafts/critic-review.md`，摘录关键结论。

#### 5.3 终稿全文展示

```

📄 团队模式终稿
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

{读取并展示终稿全文}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 5.4 单人 vs 团队对比

用表格对比同一素材下两种模式的差异：

```
┌──────────┬──────────────┬──────────────┐
│          │ 单人模式      │ 团队模式      │
├──────────┼──────────────┼──────────────┤
│ Agent 数  │ 1            │ 4            │
│ 视角      │ 单一         │ 多视角碰撞    │
│ 风格      │ 统一         │ 对抗+综合     │
│ 质量把控  │ 自我检查     │ Critic 评审   │
│ 耗时      │ 较短         │ 较长          │
└──────────┴──────────────┴──────────────┘
```

> 哪种你更喜欢？各有优劣——
> 单人模式适合快速出稿，团队模式适合追求质量。
