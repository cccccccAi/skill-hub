# Claude Code 常用操作速查手册

> 面向非开发人员（产品/运营/市场/项目经理）的日常使用指南
> 最后更新：2026-02-28

---

## 一、启动 Claude Code

### 最基础的启动

```bash
# 打开终端，直接输入
claude
```

进入交互式对话界面，可以开始和 AI 聊天。

### 带任务启动

```bash
# 一启动就告诉它要干什么
claude "帮我看看这个项目的结构"

# 直接得到答案，不进入对话（用完即走）
claude -p "帮我总结这段代码的功能"
```

### 在指定项目目录启动

```bash
# 先 cd 到你的项目目录，再启动
cd ~/my-project
claude
```

> **小贴士**：Claude Code 会自动读取当前目录的项目信息，所以在项目根目录启动效果最好。

## 二、对话恢复（终端意外关闭怎么办）

这是最常见的问题——聊到一半终端崩了、电脑休眠了、不小心关了窗口。**别慌，对话不会丢失**。

### 恢复上次对话

```bash
# 恢复最近一次对话（最常用）
claude -c

# 恢复并继续提问（一步到位）
claude -c -p "刚才说到哪了？继续"
```

### 恢复指定的对话

```bash
# 打开对话选择器（会列出所有历史对话）
claude -r

# 恢复你命名过的对话
claude -r "写周报"
```

### 对话选择器操作

| 按键 | 作用 |
|------|------|
| `↑` `↓` | 上下选择对话 |
| `Enter` | 选中并恢复 |
| `P` | 预览对话内容 |
| `R` | 重命名对话 |
| `/` | 搜索对话 |
| `Esc` | 退出选择器 |

### 给对话取名字（方便以后找到）

在对话中输入：

```
/rename 写周报
/rename Q1运营方案
```

以后就能通过名字恢复：`claude -r "写周报"`

## 三、对话中的常用命令

在对话界面中，输入 `/` 开头的命令可以执行各种操作：

### 必知命令（每天都会用）

| 命令 | 说明 | 使用场景 |
|------|------|----------|
| `/help` | 查看帮助 | 忘了命令怎么用 |
| `/clear` | 清空对话历史 | 话题聊乱了，重新开始 |
| `/compact` | 压缩对话（保留关键信息） | 聊太久上下文太长 |
| `/cost` | 查看本次花了多少 token | 关注用量 |
| `/copy` | 复制 AI 最后一条回复到剪贴板 | 想把结果粘贴到其他地方 |
| `/exit` | 退出对话 | 聊完了 |

### 项目相关

| 命令 | 说明 | 使用场景 |
|------|------|----------|
| `/init` | 初始化项目配置 | 第一次在新项目里使用 |
| `/memory` | 编辑项目记忆文件 | 让 AI 记住项目的背景信息 |

### 会话管理

| 命令 | 说明 | 使用场景 |
|------|------|----------|
| `/rename 名字` | 给当前对话命名 | 方便以后找到 |
| `/resume` | 切换到其他对话 | 同时处理多个任务 |
| `/export` | 导出对话内容 | 保存聊天记录 |
| `/export 文件名.md` | 导出到文件 | 保存到本地文件 |

### 模型和设置

| 命令 | 说明 | 使用场景 |
|------|------|----------|
| `/model` | 切换 AI 模型 | 简单任务用便宜的 haiku |
| `/config` | 打开设置 | 调整偏好 |
| `/status` | 查看当前状态 | 确认版本、模型、连接 |

## 四、快捷键速查

### 最常用的快捷键

| 快捷键 | 作用 | 记忆方法 |
|--------|------|----------|
| `Ctrl + C` | 中断 AI 回答 | **C**ancel 取消 |
| `Ctrl + D` | 退出 Claude Code | 同终端退出 |
| `Ctrl + L` | 清屏（历史还在） | 清爽一下 |
| `Esc` 按两下 | 撤回/回退对话 | 反悔键 |
| `Shift + Tab` | 切换权限模式 | 下面会讲 |

### 输入相关

| 快捷键 | 作用 |
|--------|------|
| `↑` `↓` | 翻看之前输入过的内容 |
| `Ctrl + R` | 搜索历史输入 |
| `Option + Enter`（Mac） | 换行（不发送） |
| `\` 然后 `Enter` | 换行（通用方法） |

### 引用文件

在输入框中，可以用 `@` 引用文件：

```
@src/index.js 帮我看看这个文件有什么问题
@docs/ 这个目录下的文档帮我总结一下
```

## 五、权限模式（重要！）

Claude Code 有三种权限模式，用 `Shift + Tab` 切换：

### 1. 普通模式（默认）

每次 AI 要修改文件或执行命令时，都会问你"可以吗？"

- ✅ 最安全
- ❌ 需要频繁确认，比较烦

### 2. 自动接受模式（acceptEdits）

AI 修改文件时自动同意，不再逐个确认。

```bash
# 启动时直接进入这个模式
claude --permission-mode acceptEdits
```

- ✅ 省心省力，适合信任 AI 的场景
- ❌ AI 可能改错文件

### 3. 计划模式（plan）

AI 只能看、不能改。只做分析和建议，不动你的代码。

```bash
# 启动时进入只读模式
claude --permission-mode plan
```

- ✅ 最安全，适合只想让 AI 分析但不动代码
- ❌ AI 不能帮你执行任何修改

> **建议**：日常使用先用「普通模式」，熟悉后可以切到「自动接受模式」提效。只想分析不想改代码时用「计划模式」。

## 六、跳过 Vim 编辑器

很多人碰到 Vim 会不知所措。以下方法帮你绕开它：

### 方法 1：避免触发编辑器

不按 `Ctrl + G`（这个快捷键会打开编辑器），直接在输入框里打字就好。

### 方法 2：万一进了 Vim

按以下顺序操作就能退出：

```
1. 按 Esc（确保退出编辑模式）
2. 输入 :q!（冒号 + q + 感叹号）
3. 按 Enter
```

> 记住这个口诀：**Esc → 冒号 q 叹号 → 回车**

### 方法 3：更换默认编辑器

把默认编辑器改成你熟悉的（比如 VS Code）：

```bash
# 在终端配置文件中加一行（Mac 用户编辑 ~/.zshrc）
export EDITOR="code --wait"
```

这样 `Ctrl + G` 会打开 VS Code 而不是 Vim。

## 七、直接执行终端命令

在对话中，用 `!` 开头可以直接执行终端命令（不经过 AI）：

```bash
! git status          # 查看 Git 状态
! ls                  # 查看当前目录文件
! npm run dev         # 启动开发服务器
! open .              # 打开当前目录（Mac）
```

> 这在你想快速检查某个状态但不需要 AI 分析的时候很有用。

## 八、实用工作流示例

### 场景 1：让 AI 帮忙写/改文档

```
claude
> 帮我读一下 README.md，用更通俗的语言重写一版
```

### 场景 2：分析项目结构

```
claude --permission-mode plan
> 帮我梳理这个项目的目录结构和核心文件
```

### 场景 3：日志/报错分析

```bash
# 把日志丢给 AI 分析
cat error.log | claude -p "帮我分析这个报错是什么原因"
```

### 场景 4：批量处理文件

```
claude
> 帮我把 docs/ 目录下所有的英文 README 翻译成中文
```

### 场景 5：代码审查（只看不改）

```
claude --permission-mode plan
> 帮我 review 最近一次 git commit 的改动，指出可能的问题
```

## 九、常见问题处理

### Q：启动后提示要登录？

```bash
claude auth login
# 按提示完成登录
```

### Q：不确定是否安装成功？

```bash
claude --version     # 查看版本号
claude /doctor       # 健康检查
```

### Q：想更新到最新版？

```bash
claude update
```

### Q：对话太长，AI 开始"忘事"？

```
/compact
```

这会压缩历史对话，保留关键信息，释放上下文空间。

### Q：想查看今天用了多少？

```
/cost     # 本次会话用量
/usage    # 整体额度
/stats    # 使用统计
```

### Q：误操作了想回退？

连按两次 `Esc`，可以撤回上一步操作。
